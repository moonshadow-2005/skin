# 皮肤纹理方向分析系统：项目技术介绍

## 1. 项目定位
本项目是一个面向皮肤病灶图像的分析系统，目标不是单一分割或单一检测，而是构建完整的分析流水线：

1. 病灶区域分割
2. 纹理线条提取
3. 方向与空间扇区分析
4. 局部严重度评分（Severity）
5. 存在度分级（Presence）
6. 最严重区域定位与方向统计
7. 可视化报告与交付结果整理

该系统同时支持命令行批处理和网页交互调参，适用于科研分析、病例对比和可视化演示。

---

## 2. 核心技术栈

### 2.1 深度学习与模型推理
1. PyTorch
   - 用于模型定义、权重加载和前向推理。
   - 支持 CPU/CUDA 自动切换。
2. U-Net 三分类分割
   - 主要用于将图像划分为不同皮肤组织类别（如有效区域/核心区域/背景）。
   - 分割结果作为后续纹理与方向分析的空间约束。

### 2.2 图像处理与计算机视觉
1. OpenCV
   - 图像读写、缩放、形态学操作、边缘检测、滤波、轮廓提取等。
2. NumPy
   - 像素级矩阵运算、掩膜运算、统计计算。
3. Matplotlib
   - 过程图、方向图、扇区分析图、报告图表绘制。

### 2.3 算法模块
1. Gabor 滤波器组
   - 用于增强纹理响应，突出条纹/线性结构。
2. Scharr 梯度 + 结构张量（Structure Tensor）
   - 用于局部方向估计，提取纹理主方向。
3. 方向圆统计（Circular Statistics）
   - 处理 0-180 度方向折叠问题，提高主方向与一致性计算稳定性。
4. 局部滑窗评分
   - 在圆形邻域中计算密度与方向一致性，得到局部严重度评分。

### 2.4 工程与交互
1. Streamlit 网页 Demo
   - 提供上传图片、参数调节、重算、可视化展示的一体化交互。
2. Python CLI 脚本体系
   - 支持单病例、全量批处理、模块独立运行。
3. Git 版本管理
   - 通过 .gitignore 控制模型和大规模中间产物不入库。

## 3. 技术链路（从输入到输出）

### 3.1 分割阶段
输入原图后，U-Net 输出像素级类别掩膜。该掩膜用于：
1. 定义有效分析区域
2. 约束纹理提取范围
3. 约束最严重框候选区域

### 3.2 纹理提取阶段
在有效区域内进行如下处理：
1. 形态学清理（close/dilate/erode）
2. Gabor 多方向滤波增强纹理
3. 自适应阈值 + Canny 边缘提取纹理线

输出包括：
1. 纯纹理线条图
2. 纹理与原图叠加对比图

### 3.3 方向分析阶段
基于纹理图计算梯度并构造结构张量：
1. Scharr 求导得到局部梯度
2. 结构张量特征分解获得主方向
3. 全图方向箭头可视化
4. 8 扇区统计（密度、一致性、综合分）

### 3.4 严重度与 Presence 阶段
局部严重度分数定义为：

$$
Score = w_d \cdot Density + w_c \cdot Consistency
$$

其中：
1. Density：局部纹理像素占比
2. Consistency：局部方向一致性（双角向量法）
3. $w_d, w_c$：可调权重

随后将归一化分数转为：
1. Severity Map（连续热图）
2. Presence Map（分级色块图）

Presence 支持两类分级策略：
1. threshold 模式：按固定阈值切分
2. quantile 模式：按分位数切分

### 3.5 最严重框阶段
在目标掩膜内搜索最严重区域：
1. 以框内掩膜均值严重度为目标函数
2. 支持最小覆盖率约束
3. 支持 1~3 个不重叠框
4. 计算每个框的主方向并输出箭头可视化

## 4. 关键工程设计

### 4.1 跨平台与路径兼容
1. 使用 pathlib 构建路径，减少绝对路径依赖。
2. 对中文路径采用 np.fromfile + cv2.imdecode 读图策略。
3. 统一项目根目录解析，支持不同机器不同目录结构。

### 4.2 参数化设计
系统将关键参数显式暴露，方便实验调优：
1. Heatmap 半径（动态/固定）
2. 纹理阈值
3. 密度/一致性权重
4. Presence 分级数与分界线
5. 最严重框尺寸、数量、最小覆盖率

### 4.3 结果组织
输出分层组织，兼顾调试与交付：
1. 中间产物目录（skin_output, predict_output, heatmap_output 等）
2. 标准结果目录 results/<id>
3. 精简交付目录 final_results/<id>

---

## 5. 网页 Demo 技术点

网页端采用 Streamlit，形成“算法 + 可视化 + 交互”的闭环：
1. 上传图像后自动触发全流程
2. 支持只重算热图/最严重框，提升调参效率
3. 所有参数附中文解释
4. 所有图像单图单行展示并带中文说明
5. Presence 支持阈值/分位双模式

这使得非算法开发人员也能快速理解参数含义和结果变化。

## 6. 可靠性与可维护性

### 6.1 健壮性策略
1. 关键输入文件存在性检查
2. 结果路径自动创建
3. 多命名规则兼容（如方向图文件后缀）

### 6.2 代码结构
1. 功能按脚本职责拆分（分割、纹理、方向、热图、最严重框、结果构建）
2. CLI 和 Web 两种入口并存
3. 关键算法函数可复用，避免重复实现

## 7. 可扩展方向

1. 模型层
   - 增加更强分割骨干（如 UNet++、Swin-UNet）。
2. 特征层
   - 引入多尺度纹理描述（LBP/GLCM/频域特征）。
3. 评分层
   - 学习型融合权重替代手工权重。
4. 产品层
   - 增加参数快照、实验对比、批量报告导出。
5. 部署层
   - Docker 化、服务化 API、GPU 推理加速。

---

## 8. 总结
本项目的技术价值在于把深度学习分割、传统视觉纹理分析、方向统计和可解释可视化整合为一个可调、可批处理、可交互的完整系统。它不仅能输出结果，还能解释结果生成过程，适合科研迭代和临床沟通展示。
